# UI 風格參考指南（Liquid Glass × Digital，本專案精簡版）

> 精簡、調整自使用者提供的《風格美術指南：Liquid Glass × Digital》，原始全文備份於
> `design-system/style-guide-liquid-glass-digital-source.md`。這份文件**只保留跟本專案
> 實際相關、可執行的部分**，並對照 `resources/styles.css` 目前的實際狀態標註「已符合」
> 或「有落差」——是給**未來新增/修改 UI 時**參考的決策依據，不是要求現在回頭重寫既有畫面。

## 核心概念（保留）

- **液態玻璃 = 外殼/互動層，終端機理性 = 內容/結構層**：玻璃質感只用在互動元件（按鈕、
  輸入框、卡片、導覽），不是整個背景都套模糊。
- 深度處理優先用「邊框 + 背景色階」而非陰影；陰影**只保留給玻璃元件**做出細緻的玻璃厚度
  高光，不要誇張反光動畫。
- 單一強調色，避免多色系統稀釋克制感；語意色（danger/success）只用在明確的狀態，不是裝飾。

## 目前專案的實際狀態（對照 `resources/styles.css`）

- ✅ **色彩系統已符合**：暖色階近黑/近白背景（`--bg`、`--surface-0..3`）、單一 `--accent`
  （`#5b7fff` 深色 / `#4060e8` 淺色）、`--danger`/`--success` 僅用於狀態顯示，跟指南建議
  一致，**不需要改**。
- ⚠️ **字體是混合策略，不是純等寬**：目前 Inter（一般 UI 文字）+ JetBrains Mono（數值、
  mono-label、terminal log）分工使用，跟指南「opencode.ai 全站單一等寬字體」的建議不同。
  這是既有、刻意的選擇，**不必回頭改**——但新增文字量大的區塊時，優先沿用現有的
  Inter/Mono 分工邏輯，不要無故引入第三種字體。
- ⚠️ **圓角不統一**：現況散落在 2px/3px/4px/6px/7px/10px/16px/50%（圓形）多種值，跟指南
  「避免圓角策略不一致」的警告正好相反。**新增元件時**：非玻璃元件盡量落在既有的
  4–8px 區間，不要發明新的圓角值；玻璃元件維持 `--glass-radius`（10px）。
- ⚠️ **陰影用在很多非玻璃元件上**（一般按鈕、滑桿 thumb 都有 `box-shadow`），跟指南「扁平
  元素不用陰影」的建議不同。**新增元件時**：優先用邊框/背景色階做層次，除非是玻璃元件，
  否則不要加新的 `box-shadow`。

## 未來新增 UI 時的具體原則

1. **新玻璃元件**：`backdrop-filter: blur(...)` 中等強度即可，用 `--glass-border-hi` 做
   細緻高光邊，不要整個背景套模糊。
2. **新的扁平/資訊型元件**（狀態列、清單、表格）：邊框 + 色階分層，圓角落在既有的
   4–8px 區間，不加新的陰影。
3. **條列/狀態符號**：新增「純資訊型」清單時，可以考慮 `[+]` `[-]` `[x]` 這類 ASCII 風格
   符號取代圖形 icon（這不是要把現有 SVG icon 全部換掉，只是給新清單一個可選方向）。
4. **動效**：玻璃元件可以有輕微模糊過渡；其餘元素動效簡短、線性，不要彈跳/誇張緩動——
   現有 `.modal-content` 的 cubic-bezier 過場已經算克制，維持這個尺度即可。
5. **強調色**：任何新狀態/按鈕優先重用 `--accent`，不要為了區分功能而加新色系。

## 不適用於本專案的部分（來源指南有提到，刻意略過）

- cobalt.tools 式的「單一大型膠囊輸入框佔滿視覺焦點」——本專案是多面板的檔案轉換工具
  （檔案列表 + 設定面板），沒有這種單一輸入焦點的版面邏輯，不套用。
- 吉祥物/插圖角色點綴空狀態——超出目前產品調性，不採用。
- 全小寫、口語化文字語氣——介面文字維持現有的一般大小寫（Sentence case），不跟進
  cobalt.tools 的語氣。
